# Демо для стоматолога: сценарий показа

Цель — за 10 минут показать живую систему: бот сам записывает, напоминает,
дружит с календарём, отвечает на бытовые вопросы и отчитывается деньгами.
Администратора он дёргает только когда пациент сам попросил человека.
Всё работает с ноутбука, деплой на сервер ничего не меняет, кроме адреса.

## Подготовка (за час до встречи)

1. Поднять базу и проверить систему:

   ```powershell
   docker compose up -d
   python -m navbat.onboard --demo          # демо-клиника + токен из .env (идемпотентно)
   python -m navbat.onboard --clinic 00000000-0000-4000-8000-000000000d31 --address "Ташкент, ул. Навои, 10"
   python -m navbat.onboard --clinic 00000000-0000-4000-8000-000000000d31 --payment "Наличные, карта, Payme. Рассрочки нет" --phone "+998 71 200-00-00"
   python -m navbat --check                 # все строки должны быть [OK]
   ```

2. Один раз настроить бота (если ещё нет):
   - @BotFather → `/newbot` → токен;
   - свой chat id — написать @userinfobot;
   - `python -m navbat.onboard --clinic 00000000-0000-4000-8000-000000000d31
     --tg-token <токен> --admin-chat <ваш chat id>`.

3. (Опционально, сильный ход) Google Calendar:
   - `python -m navbat.calendar.auth --clinic <id>` — авторизация Google;
   - `python -m navbat.onboard --clinic <id> --list` — взять id врача;
   - `python -m navbat.onboard --clinic <id> --doctor <id> --calendar <календарь>`.

4. Запуск на время демо (короткие напоминания, живой NLU — копейки):

   ```powershell
   python -m navbat --real --reminder-offsets 4,2
   ```

5. Сбросить следы репетиции: отменить тестовые записи через бота или
   пересоздать демо-клинику (дроп строк руками не нужен — бот сам ведёт учёт).

## Сценарий (телефон стоматолога или ваш второй аккаунт)

| 3 | Жмёте 🟢-день → весь день сеткой времени → выбираете слот | Слоты настоящие: график врачей, обед, буфер между приёмами. Два пациента один слот не возьмут никогда |

| 9 | (Если настроен GCal) Открываете Google Calendar — запись уже там. Создаёте руками событие поверх другой записи бота | Живая запись главнее: бот сам переносит своего пациента на ближайшее время и извиняется. Никаких накладок |

## Чеклист онбординга — 5 тест-диалогов

Прогоняется ПЕРЕД передачей бота живой клинике (после онбординга, с её
ботом, прайсом и адресом). Каждый пункт должен пройти без эскалации:

1. **Запись на узбекском**: «ertaga tish tozalashga yozilsam bo'ladimi» →
   слоты → выбор → имя → телефон кнопкой → подтверждение. Проверить
   запись в `--list`/календаре.
2. **Календарь**: «Другое время» → «📅 Выбрать дату» → 🟢-дни совпадают
   с графиком врачей, выходной пуст; выбор дня → сетка времени → запись.
3. **Бытовые вопросы**: «сколько стоит отбеливание?», «manzil?»,
   «до скольки работаете?», «рассрочка есть?» → цена/адрес/часы/оплата
   без эскалации; кнопка «ℹ️ О клинике» → карточка только с заполненными
   полями; «а в 21 можно?» → строка про рабочие часы.
4. **Перенос и отмена**: «перенесите мою запись на послезавтра» → новые
   слоты → подтверждение; отмена кнопкой из напоминания → слот освобождён.
5. **Админ-чат**: `/stats 7` показывает деньги недели; ерунда от
   пациента → «не понял» с кнопкой «👤 Позвать администратора», тап по
   ней → алерт + `/release <chat>` возвращает бота (то же — текстом
   «позовите администратора»); `/dayoff 15.06 Ремонт` — день закрыт,
   `/dayopen 15.06` — открыт.

## Если что-то пошло не так

- Бот молчит → `python -m navbat --check`; чаще всего не поднят Docker.
- «Не понял» на нормальной фразе → запущено без `--real`
  (фикстурный NLU знает только тестовые фразы).
- Напоминание не пришло → офсеты считаются от времени записи: запись должна
  быть дальше в будущем, чем офсет (для `--reminder-offsets 4,2` — минимум
  через 4 минуты).
- «Где вы находитесь?» отвечает меню → не задан адрес
  (`onboard --address`).
- В карточке «ℹ️ О клинике» нет оплаты или телефона → поля не заданы
  (`onboard --payment` / `--phone`; пустое значение очищает поле).

## Что говорить про деньги

- Бот записывает, пока клиника спит: в `/stats 7` видно, сколько записей
  пришло вне рабочих часов — это пациенты, которых раньше забирал
  автоответчик конкурента.
- Неявки: напоминание с кнопкой отмены освобождает слот заранее — в
  сводке это «предотвращено неявок + сумма». Одна спасённая запись
  окупает месяц обслуживания.
- «Эскалаций: 0» — администратора бот дёргает только по прямой просьбе
  пациента: словами или кнопкой «👤 Позвать администратора». Пациент
  никогда не в тупике — до живого человека всегда один тап. Всё
  остальное (наличие, цены, адрес, часы, оплата, телефон, перенос,
  отмена) бот закрывает сам.
- /stats — не просто счётчик: владелец видит динамику (тренды к прошлому
  периоду), кто из врачей приносит деньги и какая услуга — хит. Это
  аргумент «вы управляете клиникой по цифрам», а не «у вас есть бот».
- Себестоимость ИИ: ~30–100 тыс сум/мес на клинику — копейки против одной
  потерянной записи.
- Модель: разовая настройка + ежемесячное обслуживание («мы держим и
  обслуживаем вашего администратора»).
- Дальше: пилот 2 недели бесплатно → смотрим цифры /stats вместе.
