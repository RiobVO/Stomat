# Демо для стоматолога: сценарий показа

Цель — за 10 минут показать живую систему: бот сам записывает, напоминает,
дружит с календарём и отчитывается цифрами. Всё работает с ноутбука,
деплой на сервер ничего не меняет, кроме адреса.

## Подготовка (за час до встречи)

1. Поднять базу и проверить систему:

   ```powershell
   docker compose up -d
   python -m navbat.onboard --demo          # демо-клиника + токен из .env (идемпотентно)
   python -m navbat --check                 # все строки должны быть [OK]
   ```

2. Один раз настроить бота (если ещё нет):
   - @BotFather → `/newbot` → токен;
   - свой chat id — написать @userinfobot;
   - `python -m navbat.onboard --clinic 00000000-0000-4000-8000-000000000d31
     --tg-token <токен> --admin-chat <ваш chat id>`.

3. (Опционально, сильный ход) Google Calendar:
   - `python -m navbat.calendar.auth --clinic <id>` — авторизация Google;
   - `python -m navbat.onboard --clinic <id> --list` — взять id врача;
   - `python -m navbat.onboard --clinic <id> --doctor <id> --calendar <календарь>`.

4. Запуск на время демо (короткие напоминания, живой NLU — копейки):

   ```powershell
   python -m navbat --real --reminder-offsets 4,2
   ```

5. Сбросить следы репетиции: отменить тестовые записи через бота или
   пересоздать демо-клинику (дроп строк руками не нужен — бот сам ведёт учёт).

## Сценарий (телефон стоматолога или ваш второй аккаунт)

| Шаг | Что делаете | Что проговариваете |
|---|---|---|
| 1 | Пишете боту: «алло здравствуйте мне бы на чистку завтра после обеда» (можно с опечатками, можно по-узбекски: «ertaga tish tozalashga yozilsam bo'ladimi») | Понимает узбекский, русский и суржик — как пишут реальные пациенты. Это работает в 3 часа ночи, когда администратор спит |
| 2 | Бот предлагает свободные слоты кнопками — жмёте время | Слоты настоящие: график врачей, обед, буфер между приёмами. Два пациента один слот не возьмут никогда |
| 3 | Бот спрашивает имя, затем телефон — кнопкой «📱 Отправить мой номер» (одно нажатие) → запись подтверждена | Номер берётся прямо из Telegram: пациент не ошибётся в цифрах, чужой контакт бот не примет. Телефон хранится в зашифрованном виде, в интернет имена не уходят |
| 4 | Спрашиваете: «а сколько стоит отбеливание?» | Цены — из вашего прайса. Вопрос посреди записи не сбивает диалог |
| 5 | Ждёте 2 минуты → приходит напоминание с кнопками «Приду» / «Отменить» | В жизни это за 24 часа и за 2 часа. Кнопка «Отменить» освобождает слот сразу — вместо неявки вы получаете свободное время, которое успеете продать |
| 6 | (Если настроен GCal) Открываете Google Calendar — запись уже там. Создаёте руками событие поверх другой записи бота | Живая запись главнее: бот сам переносит своего пациента на ближайшее время и извиняется. Никаких накладок |
| 7 | Со своего (админского) аккаунта: `/stats` | Сводка дня: записи, отмены, напоминания. Каждый вечер приходит сама |
| 8 | Пишете боту ерунду («ыыыыы») дважды | Бот не выдумывает: честно передаёт администратору, а вам прилетает уведомление с контекстом |

## Если что-то пошло не так

- Бот молчит → `python -m navbat --check`; чаще всего не поднят Docker.
- «Передаю администратору» на нормальной фразе → запущено без `--real`
  (фикстурный NLU знает только тестовые фразы).
- Напоминание не пришло → офсеты считаются от времени записи: запись должна
  быть дальше в будущем, чем офсет (для `--reminder-offsets 4,2` — минимум
  через 4 минуты).

## Что говорить про деньги

- Себестоимость ИИ: ~30–100 тыс сум/мес на клинику — копейки против одной
  потерянной записи.
- Модель: разовая настройка + ежемесячное обслуживание («мы держим и
  обслуживаем вашего администратора»).
- Дальше: пилот 2 недели бесплатно → смотрим цифры /stats вместе.
